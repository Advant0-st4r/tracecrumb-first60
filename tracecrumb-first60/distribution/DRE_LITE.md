# TraceCrumb First-60 — DRE Lite Checklist

## T-3

- Confirm the MVP reaches first value in ≤5 minutes.
- Prepare 3 realistic examples.
- Prepare one copyable artifact output.
- Confirm the hook is understandable in 5 seconds.

## T-2

- Find 5–10 active pain threads in: r/sre, r/devops, SRE Slack, HN after manual signals, X pain replies.
- Save exact pain language.
- Select only threads where the pain is active, not abstract.
- Prepare 3 reply variants.

## T-1

- Test live URL.
- Test auth path.
- Test no-auth or demo path if available.
- Test bad input.
- Test AI unavailable fallback.
- Test copy/export.

## T-0

- Drop into 3–5 exact pain threads only.
- Use one behavior-specific ask: Run one old incident and answer: would this have changed your first diagnostic branch?
- Log behavior. Ignore likes.
- Reply within 2 hours if someone responds.

## Non-Negotiables

- Do not broadcast.
- Do not pitch the whole TraceCrumb umbrella.
- Do not explain architecture first.
- Do not monetize in this phase.
- Do not scale before own-case tests.
- Do not treat compliments as validation.

## Top Risks + Anticompounders

### Risk 1 — Generic positioning
**Anticompounder:** Use the wedge: Wrong-first-call loss prevention.

### Risk 2 — Wrong audience engagement
**Anticompounder:** Only post where the exact pain moment appears.

### Risk 3 — First-value friction
**Anticompounder:** Add demo/seed example and copyable output before posting.
